# features/

This directory will contain the Gherkin feature files for WISPGen.

**Feature files are spec.** They may only be created or modified with explicit human approval. See `constitution.md` and `AGENTS.md` for the rule.

When this project reaches the point where scenarios need to be committed, use the inventory in `docs/bdd/feature-index.md` as the source of truth. Each `.feature` file corresponds to one entry in that table.

Planned feature files:
- `signup-and-onboarding.feature`
- `authentication.feature`
- `user-and-role-management.feature`
- `domain-seeding-and-questions.feature`
- `domain-assignment.feature`
- `contributor-questionnaire.feature`
- `review-workflow.feature`
- `wisp-versioning-and-export.feature`

Do not create these files without approval.
