# Infrastructure

Terraform, nginx, certbot, systemd, and deployment artifacts for WISPGen.

## Architecture

- Single EC2 `t3.large` in `us-east-1` with an encrypted gp3 root volume.
- Elastic IP attached to the instance.
- nginx serves the React static build and proxies `/api`, `/docs`, `/redoc`, `/openapi.json` to uvicorn on localhost:8000.
- certbot DNS-01 wildcard certificate for `*.app.wisp.llc`.
- Backend runs as a systemd service under a dedicated `wispgen` user.
- Tenant databases live in `/srv/wispgen/data/tenants/`.
- Nightly S3 backup of the data directory via systemd timer.

## Prerequisites

- AWS CLI configured with a personal profile (`~/.aws/credentials`).
- Terraform >= 1.5.
- SSH client.
- Your current public IPv4 address for the SSH allowlist.

## Initial setup

1. Move into the infra directory:

```bash
cd infra
```

2. Initialize Terraform:

```bash
terraform init
```

3. Plan with your SSH CIDR (replace with your actual public IP):

```bash
terraform plan -var="allowed_ssh_cidr=24.11.224.55/32"
```

4. Apply when ready:

```bash
terraform apply -var="allowed_ssh_cidr=24.11.224.55/32"
```

5. Note the elastic IP output:

```bash
terraform output public_ip
```

6. Create Route 53 records:
   - Apex A record: `app.wisp.llc` → elastic IP
   - Wildcard A record: `*.app.wisp.llc` → elastic IP

7. Obtain the wildcard certificate once DNS has propagated:

```bash
ssh -i ../.keys/wispgen-deploy.pem ubuntu@$(terraform output -raw public_ip)
sudo certbot certonly --dns-route53 -d "app.wisp.llc" -d "*.app.wisp.llc"
```

8. Populate the environment file and deploy:

```bash
scp -i ../.keys/wispgen-deploy.pem infra/files/env.template \
  ubuntu@$(terraform output -raw public_ip):/srv/wispgen/env/wispgen.env
# Edit the env file on the server, then run:
HOST=$(terraform output -raw public_ip) ./scripts/deploy.sh
```

## Post-deploy checks

- EBS root volume is encrypted: verify in the EC2 console or with `aws ec2 describe-volumes`.
- Security group allows only the provided CIDR on port 22 and the world on 80/443.
- Wildcard cert covers `*.app.wisp.llc`: `openssl s_client -connect demo.app.wisp.llc:443 -servername demo.app.wisp.llc`.
- HTTPS response from demo tenant returns 200: `curl -fsSI https://demo.app.wisp.llc/`.

## Operational notes

- The SSH private key is generated by Terraform and saved to `.keys/wispgen-deploy.pem`.
- Rotate the SSH allowlist by changing `allowed_ssh_cidr` and re-running `terraform apply`.
- Backups run nightly to the S3 bucket created by Terraform; `WISPGEN_BACKUP_BUCKET` must be set in `/srv/wispgen/env/wispgen.env`.
- The `scripts/deploy.sh` script pulls the latest `main` branch on the server, rebuilds the frontend locally, syncs it, installs backend dependencies, and restarts services.

## Files

| File | Purpose |
|------|---------|
| `provider.tf`, `variables.tf`, `main.tf`, `outputs.tf` | AWS infrastructure |
| `userdata.sh` | EC2 bootstrap script |
| `files/nginx-wispgen.conf` | nginx wildcard vhost |
| `files/wispgen.service` | systemd unit for uvicorn |
| `files/wispgen-backup.{service,timer,sh}` | nightly S3 backup |
| `files/env.template` | server environment template |
| `Dockerfile` | CI/local parity image (production uses systemd directly) |

certbot with the Route 53 plugin should renew automatically. Verify with:

```bash
sudo certbot renew --dry-run
```
